# Deploy to AWS

This branch contains AWS CDK infrastructure generated by Armadillo. Follow these steps to deploy it to your AWS account.

## 1. Prerequisites

Verify the following tools are installed:

- **Node.js** >= 18 (`node --version`)
- **AWS CLI** >= 2.32.0 (`aws --version`)
- **AWS CDK CLI** (`npx cdk --version`, or install globally: `npm install -g aws-cdk`)

If any are missing, install them before proceeding.

## 2. AWS Authentication

Log in with your AWS Builder ID:

```sh
aws login --profile armadillo-ohau8bz0 --region us-east-1
```

This opens a browser window. Sign in with your Builder ID (the same one you used with Armadillo), then select your AWS account and role when prompted.

> Note: This command opens your browser for authentication. Complete the sign-in, then return here.

After signing in, verify:

```sh
aws sts get-caller-identity --profile armadillo-ohau8bz0
```

Expected output: your AWS account ID, IAM identity, and ARN.

Set the profile as default for this session:

```sh
export AWS_PROFILE=armadillo-ohau8bz0
```

### If `aws login` fails (no AWS account or IAM Identity Center not configured)

Builder ID and an AWS account are separate things. To use `aws login`, you need:
1. An AWS account — [create one here](https://portal.aws.amazon.com/billing/signup) if you don't have one
2. IAM Identity Center enabled on that account — [enable it here](https://console.aws.amazon.com/singlesignon)
3. Your Builder ID enrolled as a user in IAM Identity Center

If you prefer to use IAM access keys instead:

```sh
aws configure --profile armadillo-ohau8bz0
```

Enter your Access Key ID, Secret Access Key, and region (default: us-east-1) when prompted.

## 3. Bootstrap CDK (first-time only)

If this is your first CDK deployment to this account/region, bootstrap:

```sh
cd infra
npx cdk bootstrap
```

If you see "Environment aws://ACCOUNT/REGION is already bootstrapped", skip this step.

## 4. Deploy

```sh
cd infra
npm install
npx cdk deploy --all --require-approval broadening --progress events
```

Review any IAM or security-related changes when prompted, then approve.

## 5. Verify

After deployment completes, CDK outputs will display:

- `FrontendStack-synthetic-supabase-app-1779291286.DistributionDomainName` — your CloudFront URL
- `FrontendStack-synthetic-supabase-app-1779291286.BucketName` — your S3 bucket
- `FrontendStack-synthetic-supabase-app-1779291286.DistributionId` — your CloudFront distribution ID

Open the CloudFront distribution URL in a browser to verify your site is live. Allow 5-10 minutes for initial CloudFront propagation.

## 6. Teardown (if needed)

To remove all deployed resources:

```sh
cd infra
npx cdk destroy --all
```

## Troubleshooting

### "aws login" fails or times out

- Ensure AWS CLI >= 2.32.0: `aws --version`
- Check network connectivity
- Try: `aws login --profile armadillo-ohau8bz0 --region us-east-1 --no-browser` for manual device code flow
- If you don't have IAM Identity Center: use `aws configure --profile armadillo-ohau8bz0` with IAM access keys instead

### Credentials expired during deploy

Re-authenticate: `aws login --profile armadillo-ohau8bz0 --region us-east-1`

### CDK bootstrap error

- "Access Denied": Your IAM identity needs `cloudformation:*`, `s3:*`, `iam:*` permissions. Use an admin role or attach the `AdministratorAccess` policy for initial setup.
- "already bootstrapped": Safe to skip.

### "No stacks found" or wrong stack deployed

Verify `infra/bin/infra.ts` instantiates the correct stack. The stack name should be `FrontendStack-synthetic-supabase-app-1779291286`.

### Deploy fails mid-way (partial failure)

CDK automatically rolls back failed stacks — you do not need to run a manual rollback. Read the error message from the CloudFormation events output, fix the underlying issue (permissions, resource limits, invalid configuration), and re-run `npx cdk deploy --all --require-approval broadening --progress events`. CDK is idempotent — re-deploying after a fix is safe.

### Deploy fails with "Resource already exists"

Another deployment may have created resources with the same name. Each Armadillo deployment uses a unique suffix (`synthetic-supabase-app-1779291286`), so this typically means a previous partial deploy. Run `npx cdk destroy --all` and retry.

### CloudFront returns 403

- Wait 5-10 minutes for distribution propagation
- Verify the S3 bucket contains built assets in the `dist/` directory
- Check the OAC (Origin Access Control) is attached to the distribution

### Node.js version mismatch

CDK requires Node.js 18+. Check: `node --version`. If using nvm: `nvm use 18`

### Permission errors during deploy

Your IAM identity needs permissions for CloudFormation, S3, CloudFront, IAM, and Lambda. For the simplest path, use an identity with `AdministratorAccess`. For least-privilege, the minimum required managed policies are:
- `AWSCloudFormationFullAccess`
- `AmazonS3FullAccess`
- `CloudFrontFullAccess`
- `IAMFullAccess` (needed for OAC and bucket policies)
